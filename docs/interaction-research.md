调研日期：2026-07-15。以下项目均通过其公开 GitHub README 与功能说明进行只读调研。

| 项目 | 借鉴点 | CS JobMate 落地 |
| --- | --- | --- |
| [Gsync/jobsync](https://github.com/Gsync/jobsync) | AI 简历审阅、岗位匹配、任务与投递数据在同一求职旅程中呈现；岗位由用户确认后进入跟踪器 | 保留报告的来源和证据，将 A/B/C 岗位一键加入投递管道，技能缺口转为带成果证据的任务 |
| [santifer/career-ops](https://github.com/santifer/career-ops) | 用户直接粘贴岗位 URL/描述，系统识别输入并选择对应工作流；评价结果落到跟踪器 | 新增“AI 求职助手”对话式入口，自动识别简历、JD、岗位方向和城市，再触发匹配或市场智能体 |
| [Magic-Resume/Magic-Resume](https://github.com/Magic-Resume/Magic-Resume) | 简历构建、优化、分析和岗位匹配并非孤立页面，而是同一职业决策流程 | 已保存的简历版本会关联分析报告、岗位目标和行动任务，避免分析结果脱离版本上下文 |

## 设计原则

1. **一条输入优先**：用户先表达目标或粘贴已有材料，系统识别上下文；传统表单作为补充入口而非主入口。
2. **AI 在动作前后出现**：输入阶段负责理解材料，决策阶段给出证据和选择，执行阶段将结论转为岗位、任务和证据，而不是只生成长报告。
3. **先审阅后写入**：智能体不能直接修改简历或标记投递。用户确认后，才保存简历版本、岗位状态或成果。
4. **每次分析有下一步**：JD 匹配可以生成成长任务；市场分析可以进入投递管道；完成任务后再用新简历版本评估。

## 后续需求建议

- 支持粘贴岗位链接后提取页面 JD，并向用户展示来源与提取片段后再分析。
- 增加“本周三件事”智能排序，结合截止日、岗位等级和技能任务收益生成每日推荐。
- 在岗位详情中增加“为这份 JD 定制简历”对话分支，输出可审阅的修改建议而非直接覆盖版本。
- 增加面试复盘对话：用户用自然语言描述面试过程，系统结构化为题目、薄弱主题和后续任务。
